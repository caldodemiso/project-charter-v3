🔌 project overview

▶

homelab for cybersecurity & it systems practice

This project focuses on designing and building a personal Homelab environment to gain hands-on experience with IT systems, networking, and cybersecurity. The Homelab integrates smart home devices, segmented networks, monitoring tools, and vulnerability labs to simulate real-world scenarios in a safe, self-contained setup.

🎯 academic excellence: strengthen technical skills through practical application of classroom concepts
💼 portfolio development: build a portfolio-ready showcase of IT and cybersecurity capabilities
🔬 experimentation platform: provide a flexible, scalable testbed for experimenting with new tools and configurations

🌐 vision & strategic alignment

▶
🎯 mission (present): grow as an IT and cybersecurity professional by building hands-on experience, applying continuous learning, problem-solving, and documentation.
🔮 vision (future): work in cybersecurity or IT systems administration, using technical expertise to protect organizations and improve system reliability — with a portfolio-ready showcase of skills.
⚡ strategic alignment:
  • academic: applies project management and IT knowledge from coursework
  • career: produces portfolio-ready evidence of technical skills
  • personal: builds a scalable home environment for ongoing experimentation

💼 business case & justification

▶

This homelab directly advances both academic and career goals:

🎓 academic value: provides a practical project to apply IT project management concepts, including scope, stakeholders, dependencies, and deliverables
💼 career value: creates a professional portfolio artifact demonstrating cybersecurity, networking, and system administration skills
👤 personal value: improves technical competence with real-world tools and environments in a low-risk, self-paced setup

📋 project scope

▶
✅ in scope for semester
integrate at least one HomeKit smart device (light/thermostat)
configure a segmented test network with VLANs and at least 2 VMs (firewall + client)
deploy a self-hosted Minecraft server with user access and backups
implement monitoring with Grafana, Prometheus, or Uptime Kuma
deploy a Vulnerability Lab (DVWA / Metasploit) and run a basic attack/defense exercise
document configurations, testing results, and lessons learned
🔘 roadmap items (≤6 months)
add more smart home devices and expand automation
implement a SIEM tool for monitoring and logging
test additional intrusion detection/prevention tools (IDS/IPS)
experiment with cloud-hosted extensions (AWS, Azure)
create a portfolio write-up showcasing Homelab configurations
simulate more advanced attack/defense scenarios for deeper cybersecurity practice
❌ out of scope
full enterprise-level smart home or corporate network setup
24/7 uptime or production-grade availability
advanced penetration testing on external networks
integration with unsupported or legacy devices
professional red-team/blue-team exercises requiring multiple participants

📊 project methodology

▶
⚙️

chosen methodology: hybrid – iterative

why hybrid? some deliverables are clearly defined, others (like attack/defense practice) may evolve
why iterative? each deliverable can be built in a short cycle: plan → build → test → document → review
cadence: multiple releases over the semester (each deliverable completed, tested, and documented)

justification: predictive is too rigid, agile too unstructured; hybrid–iterative balances structure with flexibility.
